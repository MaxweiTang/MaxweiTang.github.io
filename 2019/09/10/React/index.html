React 简介

安装 React

有很多种方法来安装 React

  • 第一种是通过 CDN （不推荐）

  • npm – Node 包管理工具

  • creat-react-app – Facebook 在2016年创建的一个零配置的 React 初始化组件

    • 首先先安装 creat-react-app npm install -g creat-react-app

    • 安装需要一定的时间，安装完成后就可以创建一个 React 项目了 ，执行creat-react-app <文件名>，就会在当前目录下创建一个目录。

    • 创建完成后，我们先来看看目录结构

      • |– <文件名>

        ​ |– node_modules

        ​ |– public

        ​ |– src

        ​ |– .gitgnore

        ​ |– package-lock.json

        ​ |– package.json

        ​ |– README.md

        • node_modules/：包含所有通过 npm 安装的 node 包
        • public/：包含该项目所有后构建出的产品文件（静态资源）
        • .gitgnore/：包含该项目所有不应该添加到 git 仓库中的文件和文件夹
        • package.json：包含了 node 包依赖列表和一些去其他的项目配置

    • 同时 creat-reat-app 创建的是一个 npm 项目，还可以通过其附带的几个 npm 脚本进行操作

      • 在 http://loaclhost:3000启动项目：npm start
      • 运行所有测试：npm test
      • 构建项目的产品文件：npm build

ReactDOM

  • ReactDOM 被用在 React 的入口文件 src/index.js 中

  • ReactDOM.render 会使用 JSX 来替换 HTML 中的一个 DOM 节点。

  • ReactDOM.render() 有两个参数。

    • 第一个是准备渲染的 JSX
    • 第二个是指定 React 应用在 HTML 中的放置位置。如上文中，这个位置就是 id = ‘root’ 的节点。我们可以在文件 public/index.html 中找到这个 id 属性

  • 模块热替换(HMR)

    • 使用 create-react-app 创建的项目，在修改源代码时保存后游览器会自动刷新界面。但是如果在一个正在开发的应用中，刷新页面会大大降低我们的开发效率，调试时必须等待页面加载完毕后，那么一个大的应用可能会花费很多秒的时间才能刷新完毕。这时，使用 HMR 就能避免这个问题。
    • 模板热替换（HMR）是一个帮助开发者在又怜妻重新加载的应用工具，且不需要游览器重新刷新界面。使用 create-react-app 开启 HMR，只需要在 React 的入口文件 src/index.js 中添加一点配置文件即可
    • 使用 HMR 最大的好处就是能够保持应用的状态

React 基础

组件内部状态（局部状态）

  • 组建内部状态也成为局部状态，用于操作内部的数据。

  • 使用 ES6 类可以在函数初始化状态。

  • 构造函数只会在组件初始化时调用一次。

  • 当使用 ES6 编写的组件有一个构造函数时，它需要强制地调用 super(); 方法，因为这个 App 组件是 Component 的子类

单向数据流

受控组件

  • 不受控组件：表单元素如 <input>、<textarea>和<select>会以原生 HTML 的形式保存它们自己的状态。那么一但有人从外部做了一些修改，它们久会修改内部的值。

  • 在 React 中我们应该确保这些元素为受控组件。以上文中和表单交互为例，其中的输入框的值其实已经在 searchTerm 状态属性中保存了，我们可以从这里直接访问

  • 这样输入框的单向数据流循环是自包含的，组件内部状态是输入框唯一数据来源

拆分组件

  • 当我们有一个大型的 App 组件，它还在不断地扩展，那么最后可能会变得十分混乱。我们就可以将其拆分成若干个更小的组件

  • 以上文为例，我们可以将其拆分成一个搜索和一个展示列表的组件。我们可以给组件传递属性并在组件中使用。而 App 组件需要传递本地状态（state）托管的属性和它自己的类方法

  • 接着我们就可以定义这些组件

    • 每个组件中的 props 对象可以通过这个类实例的 this 来访问。而 props 是 properties（属性）的简写。当我们在 App 组件中使用该组件时，props 就有 App组件传递给该组件的所有值，这样组件就可以沿着组件数向下传递属性了。

  • 这些组件是通过 props 对象来获取它们的值，所以当我们在别的地方重用它时，它可以每一次都传递不同的 props，这样就可以复用这些组件了

可组合组件

  • 在组件的 props 对象中有一个：children 属性。它可以将元素从上层传到该组件中。

  • 现在 srearch 组件就可以从 props 对象中解构出 children 属性

  • children 可以将一个元素或者元素树作为子元素进行传递。其让组件相互组合一起成为可能

可复用组件

  • 之前定义的 Search 和 Table 组件也是可以复用的。现在我们来定义一个可复用的组件 Button

  • 其中在 Button 组件中 props 中有一个 className 属性，这个属性是 React 基于 HTML 属性 class 的另一个衍生物。

  • 使用复用组件可以在将来使用 button 时容易改变它们的属性、样式或行为，而不必重构每个 button

无状态组件

  • 无状态组件就是函数，它接收一个输入并返回一个输出。输入是 props，输出是一个 JSX 组件实例。

  • 那么无状态组件和 ES6 类组件有什么区别呢？

    • 无状态组件没有本地状态（无状态）。即我们无法通过 this.state 或者 this.,setState() 来访问或则和更新状态。
    • 无状态组件也没有生命周期方法。

  • 那么现在我们就可以将之前的 Search 组件重构成无状态组件
