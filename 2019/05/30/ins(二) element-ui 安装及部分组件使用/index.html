前言

界面设计方面我选择了 element-ui ,它是一套采用 Vue 2.0 作为基础框架实现的组件库，一套为开发者、设计师和产品经理准备的基于 Vue 2.0 的组件库，提供了配套设计资源，帮助网站快速成型。这篇文章主要是谈谈我在这个项目中使用 element-ui 以及 在页面中遇到的样式问题。

一、element-ui

在这里可以看见我们在创建项目时所安装的插件，点击右上角的添加插件，接着在搜索框中输入 element，结果中的第一个就是我们需要的 element-ui 插件啦，点击安装即可。

2、使用element-ui 组件

关于element-ui的使用，我认为看看其官方文档就能够满足基本的使用了。这里主要讲讲一些由于项目需要，基于其组件自定义的组件。

首先先讲讲在这个项目中我使用了那些组件。

Container

  • 布局容器搭建页面的基本结构

Layout 布局

  • 类似Bootstrap栅格系统。通过基础的24分栏，迅速简便地创建布局

该项目主要是采用Layout布局对组件在页面中的位置进行处理。主要运用到的参数是span和offset

Popover弹出框

  • 实现顶栏导航栏点击爱心图标，在该图标下弹出一指定高度的弹出框。

el-popver标签中 width指定弹出框的高度；trigger：指定弹出框的触发方式。详情可以参照官方文档Popver

而要实现在弹出框中列表实现，且超出高度的部分列表卷曲，通过鼠标滚轮来调整且隐藏滚动条的效果。首先需要在Popover弹出框中添加ul列表，并固定其高度，调整其样式。

Loading

  • 加载数据时显示动效。

    使用十分简单，只需要将v-loading = 定义好的变量添加到你期望有加载效果的容器中即可

    上述代码中的v-loading的loading变量定义在data中,默认值为true，即页面初始化的使用就启动加载效果，直到过去到数据后将loading修改为false，去除加载效果。

Upload

  • 并同时生成一个新的FormData来储存表单中的信息，以供在下一个表单中与下一个表单中的信息一起提交

Dialog

  • Dialog 对话框，可以弹出一个对话框

  • 在本项目中主要是使用 Dialog显示图片详细信息和个人页的设置信息等功能，效果如下：

  • 对于这个组件我主要说下这个项目中遇到的问题

    • 首先是关于同个页面多个Dialog如何指定单独触发其中的一个

      • 比如在个人页面中的帖子中，用户发布了许多的帖子。由于组件化的缘故，帖子详情页的触发标识符都使用的同一个标识符，点击触发Dialog，同一个页面中所有Dialog组件都会被触发，所以在页面中显示的效果只会是帖子中的最后一张的详情页，所以要对触发标识符和帖子的索引同时进行判断。

      • 其中的:visible.sync是控制Dialog开启开始关闭。而.sync修饰符主要是当一个子组件改变了一个 prop 的值时，这个变化也会同步到父组件中所绑定。而其在Dialog中的主要作用就是出发Dialog后点击Dialog之外的区域或者关闭按钮能够将用户的操作传递到父组件中，改变触发标识符的值、进而将Dialog关闭。

      • 而使用的.sync的:visible是不能够进行 &&运算的，因为子组件的变化会同步到父组件，.sync中只能有一个变量。但我们要通过触发标识符和帖子索引一起判断又该怎么修改呢？最终我采用的下面的这种写法。

      • 去除.visible的.sync修饰符，由于在Dialog组件关闭时（点击组件外区域或者点击关闭按钮）会触发:before-close函数，这个时候在触发关闭函数时将触发标识符置为false即可。

  • 第二个问题是在个人页面，自己发布的帖子中可能有其他用户的评论，通过点击别人的头像就可以跳转到其个人主页中，但此时跳转后，会默认触发其中的Dialog，效果如下：

    • 由于其中的帖子部分是个人中的一个子组件，经过排查，我认为引起的原因可能是由于在同一个页面之间跳转（个人页-个人页）只重新渲染了父组件，而对子组件和其中的Dialog标识符的值并没有重置（还处于触发状态），经过测试同一个页面之间的跳转（个人页-个人页），帖子部分子组件的Vue的生命周期钩子函数也没有触发。所以我采用了使用父子组件双向通行，使用在父组件中定义的帖子部分的Dialog触发标识符来判断其是否被触发。
